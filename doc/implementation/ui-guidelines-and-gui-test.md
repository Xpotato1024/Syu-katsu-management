# UIデザイン方針・実装ルール・GUIテスト作法

最終更新: 2026-03-06

## 1. デザイン方針（SaaS/OSSスタイル）
- 情報密度を保ちつつ、操作対象を明確に区別する。
- ブランド表現はヘッダーとアクセント色に限定し、本文領域は可読性優先で落ち着いた面構成にする。
- ステータス表現は `色 + 文言` の二重化を基本とし、色だけで判別させない。
- 角丸や影は過剰にせず、業務UIらしい安定感を優先する。

## 2. 実装ルール（フロントエンド）
### 2.1 ダークモード
- `html.dark` クラスを単一スイッチとして扱う。
- ライト固定の色値を新規追加しない。既存に追加する場合は `.dark` 側の上書きを同時に実装する。
- コンポーネントごとに以下を点検する。
  - 背景色
  - 文字色
  - 枠線色
  - 区切り線（`border-top` / `border-bottom`）
  - ホバー時の背景色
- 特に見落としやすい項目。
  - サイドバーの `button` / `a`
  - フッター
  - 編集フォーム内の `input` / `select` / `textarea`
  - テーブル・タイムラインのセル境界線

### 2.2 トースト通知
- 成功/情報/エラーは `色 + アイコン + タイトル（成功/情報/エラー）` の3点セットで区別する。
- 文言は「何が起きたか」を最短で示す（例: `企業情報を保存しました。`）。
- 閉じる操作を必ず提供し、オートクローズは既定3.2秒。

### 2.3 フッター
- 中央揃えで統一し、以下を固定表示する。
  - アプリ名
  - アプリバージョン（次回リリース予定版を記載）
  - GitHub / ライセンス / 利用規約
  - コピーライト
- バージョンは `VITE_APP_VERSION` で制御し、未設定時は `v0.3.5-next` を表示する。

### 2.4 利用規約
- 免責文はフッターへ常時露出しない。
- フッターの `利用規約` ボタンからダイアログ表示する。
- 規約改定時はこのファイルと利用規約文面を同時更新する。

### 2.5 アプリ命名ルール
- 日本語アプリ名は `就活マネジメント` を採用する。
- 英語タイトルは `Job Hunt Manager` を採用する。
- ヘッダーのタイトル付近に `by xpotato.net` を表示する。
- フッターとブラウザタイトル（`<title>`）も上記命名と整合させる。

## 3. GUIテスト作法
### 3.1 基本起動
```bash
cd /mnt/d/Xpotato-apps/Syu-katsu-management
./scripts/gui-up.sh
```

### 3.2 ポート競合時（推奨）
```bash
cd /mnt/d/Xpotato-apps/Syu-katsu-management
FRONTEND_HOST_PORT=25173 BACKEND_HOST_PORT=28080 DB_HOST_PORT=25432 ./scripts/gui-up.sh
```

### 3.3 テスト観点（最低限）
1. テーマ切替:
   - ライト/ダーク双方で背景・文字・枠線のコントラストが崩れない。
   - サイドバー、フッター、フォーム、タイムラインで明色漏れがない。
2. トースト:
   - 成功/情報/エラーで色・アイコン・タイトルが一致する。
   - 手動クローズが機能する。
3. フッター:
   - バージョン表示が中央揃えで崩れない。
   - `利用規約` ダイアログが開閉でき、内容が読める。

### 3.4 ビルド確認
```bash
cd /mnt/d/Xpotato-apps/Syu-katsu-management/frontend
docker run --rm -v "$PWD:/app" -w /app node:20-alpine sh -lc 'npm ci --no-audit --no-fund && npm run build'
```

### 3.5 終了
```bash
cd /mnt/d/Xpotato-apps/Syu-katsu-management
./scripts/gui-down.sh
```
