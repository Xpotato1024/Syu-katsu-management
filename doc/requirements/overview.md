# 要件定義（概要）

## 目的
就活中に応募した企業ごとの進捗・情報を一元管理し、次にやるべき行動を明確にする。

## 想定ユーザー
- 個人の就活生（まずは単一ユーザー利用を想定）

## 管理対象データ
- 企業名
- マイページリンク
- マイページID
- 選考フロー
- 選考状況
- ES内容
- 企業研究内容

## 機能要件（MVP）
1. 企業情報のCRUD
   - 企業情報を作成・閲覧・更新・削除できる
2. 選考フロー管理
   - 企業ごとに選考ステップ（例: エントリー, ES提出, 面接1, 面接2, 内定）を管理できる
3. 選考状況管理
   - ステータス（未着手 / 進行中 / 完了 / 不合格 / 辞退）を更新できる
4. メモ管理
   - ES内容・企業研究内容をテキストで記録できる
5. 一覧/検索
   - 企業名や選考状況で絞り込みできる

## 非機能要件（初期）
- Dockerでローカル開発可能
- バックエンドはGo、フロントエンドはTypeScript
- DBはPostgreSQL（テキスト検索・将来拡張性・Docker運用の容易さを重視）
- パブリック公開を見据え、機密値は `.env` 管理（`.env.example` を提供）
