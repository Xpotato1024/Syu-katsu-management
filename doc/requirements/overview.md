# 要件定義（概要）

## 目的
就活中に応募した企業ごとの進捗・情報を一元管理し、次にやるべき行動を明確にする。

## 想定ユーザー
- 個人の就活生
- 将来的な公開を見据え、ユーザー単位でデータ分離可能な構成

## 管理対象データ
- 企業名
- マイページリンク
- マイページID
- 選考フロー
- 選考状況
- ES内容
- 企業研究内容

## 機能要件（MVP）
1. 企業情報のCRUD
   - 企業情報を作成・閲覧・更新・削除できる
2. 選考フロー管理
   - 企業ごとに選考ステップ（例: エントリー, ES, Webテスト, GD, 面接）を管理できる
   - 企業作成時に初期ステップを登録できる
   - 選考途中で判明したステップを後から追加できる
3. 選考状況管理
   - 企業ステータス（未着手 / 選考中 / 内定 / お見送り / 辞退）を更新できる
   - ステップステータス（未着手 / 予定 / 実施済 / 通過 / 不通過 / 辞退）を更新できる
   - 各ステップに日程を後から設定・更新できる
4. メモ管理
   - ES内容・企業研究内容をテキストで記録できる
5. 一覧/検索
   - 企業名や選考状況で絞り込みできる
   - 企業を縦軸、日付を横軸にしたカレンダー表示で日程を俯瞰できる
   - 企業横断でユーザー全体の予定を日付別に一覧できる

## 非機能要件（初期）
- Dockerでローカル開発可能
- バックエンドはGo、フロントエンドはTypeScript
- DBはPostgreSQL（テキスト検索・将来拡張性・Docker運用の容易さを重視）
- パブリック公開を見据え、機密値は `.env` 管理（`.env.example` を提供）
- 認証はAuthelia OIDC連携を想定（アプリ側はユーザーIDスコープで管理）
